    SVM

    Support Vector Machine

    Introduction

    Support Vector Machine is a very famous classification algorithm (can be used in regression challenges as well) in Machine Learning, which occupied the dominant position of classification algorithms before Deep Learning. There are still many cases where SVM will be used.

    This article will explain SVM in detail, and in the end I will show the Matlab codes for implementing different SVM, which can help you to have a deeper understanding on how SVM works.

    What is SVM

    SVM is a kind of supervised machine learning algorithm, which usually used for two-classes classification.

    There are 6 key parts in SVM:

    • Margins
    • Primal Problem
    • Lagrange
    • Dual Problem
    • Soft Margin
    • Kernel

    Before we go into detail, let’s start with figuring out what exactly SVM is doing.

    The line we draw also called Hyperplane

    How can we find the optimal Hyperplane?
    This is exactly what SVM is doing, in general, SVM provides a systematic way to divide data optimal.

    How we define the “Optimal” Hyperplane?
    To answer this question, we need to understand what is “Margin”.

    Margins

    Margin，the gap that seperate two classes of data，which is the key concept in SVM。

    In general, there are three kinds of Margin：

    • Functional Margin
    • Geometric Margin
    • Margin of a training set

    we can subdivide the Margin：

    • $\gamma_{i}^{f}$ : Functional margin of an example $\mathbf({x_i, d_i})$
    • $\gamma_{i}^{g}$: Geometric margin of an example $\mathbf({x_i, d_i})$
    • $\gamma^{f}$: Functional margin of a training set
    • $\gamma^{g}$: Geometric margin of a training set
    • $\gamma$: Margin of a training set

    Let’s take a look on the mathematic definition of each kind of Margin:

    Conclusion for Margin:

    Margin is used to represent the gap that can separate two-classes data points, we want the gap as wide as possible, in order to achieve this, we calculate the Euclidean distance to each possible Hyperplane for each data point, and use the minimal Euclidean distance as the $\gamma_{i}^{g}$ under this Hyperplane, this is because we want separate two classes as accurate as possible under this Hyperplane. Then we can get the maximal over all $\gamma_{\left(\mathrm{w}, b\right)}^{g}$ we have calculated, this is because we want our model be able to tolerate noise from test set, and be robust to different data.

    The two reasons I mentioned above has some mathematical proof, however, it is not recommended that those new to SVM are caught in various incomprehensible mathematical proofs, some proof we just take it as fact.
